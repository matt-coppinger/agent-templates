# Skills Taxonomy Reference

Use this reference to determine skill equivalences and relationships when matching candidates to requirements.

## Programming Languages

### Python Ecosystem
- **Python** ≈ Python 3, CPython
- **Django** → implies Python
- **Flask** → implies Python
- **FastAPI** → implies Python
- **Celery** → implies Python, async processing

### JavaScript/TypeScript Ecosystem
- **TypeScript** → implies JavaScript
- **React** ≈ React.js ≈ ReactJS
- **Next.js** → implies React, JavaScript/TypeScript
- **Node.js** → implies JavaScript (server-side)
- **Express** ≈ Express.js → implies Node.js

### JVM Ecosystem
- **Java** ≈ Java 8+, Java 11+, Java 17+
- **Kotlin** → implies JVM familiarity
- **Spring** ≈ Spring Boot → implies Java
- **Scala** → implies JVM familiarity

## Cloud Platforms

### AWS
- **AWS** ≈ Amazon Web Services
- **EC2** → implies AWS compute
- **S3** → implies AWS storage
- **Lambda** → implies AWS serverless
- **RDS** → implies AWS managed databases
- **ECS/EKS** → implies AWS container orchestration
- **CloudFormation** → implies AWS Infrastructure as Code

### Azure
- **Azure** ≈ Microsoft Azure
- **Azure Functions** → implies Azure serverless
- **Azure DevOps** → implies CI/CD experience

### GCP
- **GCP** ≈ Google Cloud Platform
- **BigQuery** → implies GCP, data engineering

## Databases

### Relational
- **PostgreSQL** ≈ Postgres
- **MySQL** ≈ MariaDB (closely related)
- **SQL Server** ≈ MSSQL
- Any relational DB implies **SQL** proficiency

### NoSQL
- **MongoDB** → document database
- **Redis** → in-memory cache/store
- **DynamoDB** → implies AWS, NoSQL
- **Elasticsearch** → search/analytics

## DevOps & Infrastructure

### Container Orchestration
- **Kubernetes** ≈ K8s
- **Docker** → implies containerisation (prerequisite for K8s)
- **Docker Compose** → implies Docker

### Infrastructure as Code
- **Terraform** — cloud-agnostic IaC
- **CloudFormation** — AWS-specific IaC
- **Pulumi** — programmatic IaC
- **Ansible** — configuration management (related but distinct)

### CI/CD
- **GitHub Actions** → implies Git, CI/CD
- **Jenkins** → implies CI/CD
- **GitLab CI** → implies Git, CI/CD
- **CircleCI** → implies CI/CD

## API Technologies
- **REST** ≈ RESTful APIs
- **GraphQL** — distinct from REST (not interchangeable)
- **gRPC** — distinct from REST
- **OpenAPI** ≈ Swagger → implies REST API design

## Methodology & Practices
- **Agile** ≈ Scrum, Kanban (related methodologies)
- **TDD** → implies strong testing practices
- **BDD** → implies testing + requirements practices
- **CI/CD** → implies automation, DevOps awareness
- **Microservices** → implies distributed systems knowledge
